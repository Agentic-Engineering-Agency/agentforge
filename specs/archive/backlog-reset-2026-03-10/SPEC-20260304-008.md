# SPEC-20260304-008: Sessions Inspect — Accept Short ID

## Status: Active
## Priority: P0

## Objective
`agentforge sessions inspect <id>` currently requires the full `session_1772...` string.
It should also work with the Convex `_id` (the short ID shown in `sessions list`).

## Root Cause
`sessions inspect c981y6rv` passes the Convex short `_id`. The Convex query uses
`getWithMessages` which first tries `ctx.db.get(id)` — but `c981y6rv` is not a valid 
Convex ID format. It falls back to `bySessionId` index which looks for the session's 
`sessionId` field (the long string like `session_1772...`), but `c981y6rv` is the `_id`.

Fix: also look up by the Convex `_id` directly using `ctx.db.get(args.id)` where `args.id`
is treated as the Convex document ID. Or: use `byShortId` / scan for matching `_id`.

## Success Criteria
- [ ] `agentforge sessions inspect c981y6rv` (short Convex _id) works and shows session details
- [ ] `agentforge sessions inspect session_1772203723009_u9by9a8p3` (full sessionId) still works
- [ ] Output includes: sessionId, agentId, status, started, last activity, message count, last 3 messages
- [ ] Returns clear error "Session not found" for invalid IDs
- [ ] Works for completed, error, and active sessions
- [ ] CLI `packages/cli/src/commands/sessions.ts` + Convex `convex/sessions.ts` updated
- [ ] CLI dist rebuilt
- [ ] All 911+ tests pass
- [ ] Zero TypeScript errors

## Technical Details
The `getWithMessages` function in `convex/sessions.ts` should:
1. First try `ctx.db.get(args.id as Id<"sessions">)` — handles Convex document _id
2. If not found (throws or returns null), fall back to `ctx.db.query("sessions").withIndex("bySessionId", q => q.eq("sessionId", args.id)).first()`
3. The CLI `sessions inspect` command currently passes the raw string as `id` to the Convex query
