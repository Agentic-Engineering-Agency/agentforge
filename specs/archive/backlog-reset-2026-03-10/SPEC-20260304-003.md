# SPEC-20260304-003: Session Detail Dashboard View

**Status:** SPEC  
**Created:** 2026-03-04  
**Priority:** P1

## Summary

Add a `/sessions/:id` detail page to the dashboard that shows the full message history of a session, agent info, and controls (end session, export transcript).

## Requirements

### R1 — Route `/sessions/:id`
- New TanStack Router file route: `packages/web/app/routes/sessions.$sessionId.tsx`
- Loads session via `useQuery(api.sessions.get, { id: sessionId })` — create query if missing
- Loads messages via `useQuery(api.messages.getByThread, { threadId })` using session's threadId
- Back button → /sessions

### R2 — Session Header
- Agent name (link to `/agents/:id`), session ID, status badge, started/ended time, duration
- If active: "End Session" button → `useMutation(api.sessions.endSession)`
- "Export" button → downloads transcript as plain text or JSON (client-side, no backend)

### R3 — Message Thread View
- Chronological list of messages (oldest at top)
- User messages: right-aligned bubble, dark background
- Assistant messages: left-aligned bubble, lighter background, agent avatar
- Each message shows timestamp
- If no threadId: show "No message history for this session"

### R4 — Session Stats Sidebar
- Agent: name, model, provider
- Status: active/completed/error
- Started: formatted datetime
- Ended: formatted datetime (or "Active" if ongoing)
- Duration: human-readable (e.g., "2m 34s")
- Total Messages: count

### R5 — Sessions list navigation
- On `/sessions` page, each session row should link to `/sessions/:id`
- If `sessions.tsx` uses a table, make the session ID cell a link

### R6 — Convex queries needed
- `sessions.get`: `args: { id: v.id("sessions") }` — return single session by ID
- `sessions.endSession` already exists
- `messages.getByThread` already exists (added in PR #148 fixes)

## Acceptance Criteria

- `/sessions/<id>` loads with real message history ✓
- Message bubbles show user vs assistant correctly ✓
- Export button downloads transcript ✓
- End session button works for active sessions ✓
- Sessions list links to detail page ✓
- TypeScript compiles clean ✓

## Implementation Notes

- File: `packages/web/app/routes/sessions.$sessionId.tsx`
- Route registration: add to `routeTree.gen.ts` (8 locations)
- Re-use message bubble styles from `chat.tsx` for consistency
- `sessions.get` — check if it exists in `convex/sessions.ts`; add if missing
- Export: use `window.Blob` + `URL.createObjectURL` for client-side download
