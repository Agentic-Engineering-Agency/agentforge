# SPEC-20260304-004: API Token Management UI

**Status:** SPEC  
**Created:** 2026-03-04  
**Priority:** P1

## Summary

Wire up the `agentforge tokens` CLI commands and add a Tokens tab to the Settings page in the dashboard. The Convex `apiAccessTokens` module was created in PR #148 — this spec implements the UI layer.

## Requirements

### R1 — Settings page Tokens tab
- In `packages/web/app/routes/settings.tsx`, add a "Tokens" tab between "Vault" and "Advanced"
- Tab content: list of API tokens, "Generate Token" button

### R2 — Token List
- `useQuery(api.apiAccessTokens.list, {})` — shows all tokens
- Columns: Name, Token (masked: `agf_xxx...xxx`, show last 4 chars), Created, Expires, Status (Active/Revoked)
- Each row: "Revoke" button (red) → `useMutation(api.apiAccessTokens.revoke, { id })`
- If no tokens: "No tokens generated yet." empty state

### R3 — Generate Token Form
- Inline form above the list: "Token Name" text input + "Generate" button
- On submit: calls `useMutation(api.apiAccessTokens.generate, { name })`
- After generation: show a success modal/callout with the full token (only shown once)
  - "Copy Token" button, warning: "Save this token — it won't be shown again"
  - Dismiss button clears the callout

### R4 — CLI tokens command (verify/fix)
- `agentforge tokens list` — now calls `apiAccessTokens:list` ✓ (should work after schema deploy)
- `agentforge tokens generate <name>` — verify it works end-to-end
- `agentforge tokens revoke <id>` — verify it works
- Test all three against live Convex and fix any remaining issues

### R5 — Token table schema validation
- `convex/schema.ts` has `apiAccessTokens` table — verify it deployed correctly
- Run `CONVEX_DEPLOYMENT=dev:watchful-chipmunk-946 npx convex dev --once` to confirm

## Acceptance Criteria

- Settings > Tokens tab renders ✓
- Token list shows stored tokens ✓
- Generate button creates token and shows one-time reveal ✓
- Revoke button deactivates token ✓
- `agentforge tokens list` works from CLI ✓
- `agentforge tokens generate "my-token"` works from CLI ✓
- TypeScript compiles clean ✓

## Implementation Notes

- File: `packages/web/app/routes/settings.tsx`
- Radix UI Tabs already in use on this page
- Token masking: `token.slice(0, 8) + '...' + token.slice(-4)`
- "Copy to clipboard": `navigator.clipboard.writeText(token)`
- The `generate` mutation returns `{ id, token }` — use the token field for the one-time display
- convex/apiAccessTokens.ts already exists from PR #148
