# SPEC-20260304-001: CLI Enhancements

> **Spec ID:** SPEC-20260304-001
> **Status:** SPEC
> **Created:** 2026-03-04
> **Track:** Team A
> **Issue:** Fill CLI gaps discovered in full functionality audit

---

## Overview

Adds `sessions get`, `sessions delete`, thread naming, and removes window.confirm from dashboard.

---

## Problem Statement

Users report: "CLI gaps exist - missing sessions get/delete, thread rename/delete, and dashboard has window.confirm dialogs."

The audit revealed missing CLI commands and improper confirmation patterns in the web dashboard.

---

## Functional Requirements

### Section A: sessions get command shows session details with message preview

| ID | Requirement | Priority |
|---|---|---|
| A1 | Command calls Convex sessions:get query with sessionId | P1 |
| A2 | Display: session ID, agent name, status, created/ended timestamps, duration, message count | P1 |
| A3 | Command calls messages:getByThread with the session's threadId | P1 |
| A4 | Show last 5 messages as preview | P1 |
| A5 | Display error if session not found | P1 |
| A6 | Support both short IDs (last 8 chars) and full session IDs | P2 |

### Section B: sessions delete command removes sessions with confirmation

| ID | Requirement | Priority |
|---|---|---|
| B1 | Command calls Convex sessions:remove mutation with sessionId | P1 |
| B2 | Session is deleted from database | P1 |
| B3 | Display success message | P1 |
| B4 | Prompt for confirmation if --force flag not provided | P1 |
| B5 | Display error if session not found | P1 |
| B6 | Return exit code 0 on success, 1 on failure | P2 |

### Section C: threads rename command persists thread name to Convex

| ID | Requirement | Priority |
|---|---|---|
| C1 | Command calls Convex threads:update mutation with id and name | P1 |
| C2 | Thread name is updated in database | P1 |
| C3 | Display success message | P1 |
| C4 | Display error if thread not found | P1 |
| C5 | Validate that name argument is provided | P1 |
| C6 | Thread updatedAt timestamp is updated | P2 |

### Section D: threads delete command removes thread and all messages

| ID | Requirement | Priority |
|---|---|---|
| D1 | Command calls Convex threads:remove mutation with thread id | P1 |
| D2 | All messages associated with the thread are deleted | P1 |
| D3 | Thread itself is deleted from database | P1 |
| D4 | Display success message | P1 |
| D5 | Display error if thread not found | P1 |
| D6 | Prompt for confirmation unless --force flag provided | P1 |

### Section E: No window.confirm calls in packages/web

| ID | Requirement | Priority |
|---|---|---|
| E1 | agents.tsx does NOT contain window.confirm in handleDeleteAgent | P1 |
| E2 | projects.tsx does NOT contain window.confirm in handleDelete | P1 |
| E3 | Show inline "Are you sure?" prompt on card | P1 |
| E4 | Show [Delete] [Cancel] buttons inline on card | P1 |
| E5 | Clicking Delete executes the deletion | P1 |
| E6 | Clicking Cancel dismisses the confirmation state | P1 |

### Section F: agentforge logs command displays real Convex data

| ID | Requirement | Priority |
|---|---|---|
| F1 | Command displays recent session/activity logs | P1 |
| F2 | Logs are fetched from Convex | P1 |
| F3 | Each log shows timestamp, session ID, agent, status | P1 |
| F4 | Display "No logs found" if no logs exist | P2 |
