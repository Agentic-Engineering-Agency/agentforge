# SPEC-20260304-002: Dashboard Agent Detail Page

**Status:** SPEC
**Created:** 2026-03-04
**Priority:** P1

## Summary

Replace the agents page modal with a full `/agents/:id` detail page with tabs: Overview, Chat, Sessions, Settings. This matches the Manus/OpenClaw pattern of a dedicated agent workspace.

## Requirements

### R1 — Route `/agents/:id`
- New TanStack Router file route: `packages/web/app/routes/agents.$agentId.tsx`
- Loads agent via `useQuery(api.agents.get, { id: agentId })`
- Shows 404 / "Agent not found" if ID is invalid
- Back button → /agents

### R2 — Agent Header
- Agent avatar (icon based on provider), name, status badge (active/inactive)
- Toggle active/inactive button → calls `useMutation(api.agents.update, { isActive })`
- Edit button → opens settings tab

### R3 — Tabs
Three tabs:
1. **Overview** — instructions preview, model/provider, temperature, maxTokens, failover chain, sandbox config, last session time, total sessions count
2. **Sessions** — list of sessions for this agent (paginated, 10/page), each row shows session ID, status, started, duration, message count, link to session detail
3. **Settings** — full edit form for the agent (same fields as create form: name, description, instructions, provider/model, temperature, maxTokens, failoverModels, sandbox, workspace storage). Save button calls `useMutation(api.agents.update)`

### R4 — Edit Form (Settings Tab)
- Pre-populated with current agent data
- Provider dropdown triggers live model fetch via `useAction(api.modelFetcher.getModelsForProvider)`
- Save button shows loading state, calls update mutation, shows success toast

### R5 — Navigation Update
- On `/agents` page, "View Details" button → navigate to `/agents/:id` (was previously calling openModal)
- Card click → navigate to `/agents/:id`
- Keep the Create Agent modal for new agent creation only

### R6 — Sessions list for agent
- `api.sessions.listByAgent` query: `args: { agentId: v.string() }` — create this if missing in `convex/sessions.ts`
- Returns sessions for that agent ordered by createdAt desc

### R7 — Route registration
- Add new route to `routeTree.gen.ts` in all 8 required locations
- Ensure route is properly typed and accessible

## Acceptance Criteria

- `/agents/agent-test-001` loads and shows agent detail
- Three tabs render with real Convex data
- Settings tab saves changes and reflects immediately
- Sessions tab shows real sessions for the agent
- `/agents` page "View Details" navigates to detail page
- TypeScript compiles clean
- No regression in existing pages
