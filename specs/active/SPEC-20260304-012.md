# SPEC-20260304-012: Messaging Channels — Telegram & Slack Adapters Wired to Dashboard

**Status:** Active  
**Priority:** P1 — Biggest competitive gap vs OpenClaw (0 channels vs 18+)  
**Branch:** feat/overnite-c  
**Worktree:** /tmp/agentforge-overnite-c  

## Problem

AgentForge has zero messaging channels. Agents can only be accessed via the web dashboard and CLI. OpenClaw supports 18+ channels. The Connections page in the dashboard exists but has no working integrations.

The codebase already has `channel-telegram.ts` and `channel-slack.ts` adapters from Sprint 2B (PR #93). These need to be wired to:
1. The Convex backend (store channel config, route messages)
2. The dashboard Connections page (UI to configure bot token + webhook)
3. An HTTP endpoint in Convex that receives webhook events

## Acceptance Criteria

### Telegram Adapter

**Setup flow:**
1. User goes to Connections page → clicks "Add Telegram Bot"
2. Enters bot token (from BotFather)
3. AgentForge calls Telegram API to get bot info, verifies token
4. Stores config encrypted in vault (or mcpConnections table)
5. Registers webhook: `POST /api/telegram/{connectionId}`

**Message flow:**
1. User sends message to Telegram bot
2. Telegram → POST `/api/telegram/{connectionId}` (Convex HTTP endpoint)
3. Convex finds the agent linked to this connection
4. Runs `mastraIntegration:executeAgent`
5. Sends reply back via Telegram Bot API

**Convex HTTP endpoint** (`convex/http.ts`):

### Slack Adapter

**Setup flow:**
1. User clicks "Add Slack App" → redirected to Slack OAuth
2. On callback, store bot token + team info
3. Subscribe to `message` events

**Message flow:**
1. User mentions @bot or DMs the bot in Slack
2. Slack → POST `/api/slack/{connectionId}` 
3. Same pattern as Telegram

## Implementation Notes
- Bot tokens must be encrypted (use existing vault pattern from BYOK)
- Start with Telegram only (simpler than Slack OAuth), then Slack
- The Convex HTTP endpoint must verify Telegram's secret token header
- Use `fetch()` to send Telegram replies (no SDK — keep it lightweight)
- Webhook URL format: `https://watchful-chipmunk-946.convex.site/telegram/{connectionId}`
- The dashboard should show the webhook URL after setup so user can configure Telegram
- Do NOT store bot tokens in plain text — use the existing encrypt/decrypt vault functions
