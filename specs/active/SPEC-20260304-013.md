# SPEC-20260304-013: CLI CRUD Fixes + Template Drift

**Status:** Active
**Priority:** P0
**Branch:** feat/overnite-d

## Problems

### 1. `agents edit` is broken
`agentforge agents edit agent-test-001 --description "updated"` → "unknown option --description"
Expected: updates the agent in Convex and confirms.

### 2. `tokens create` / `tokens delete` missing
Both subcommands don't exist. Only `tokens list` works.

### 3. Template drift
Root `convex/` has diverged from `packages/cli/templates/default/convex/` and `packages/cli/dist/default/convex/`.
Key drift:
- `schema.ts`: templates missing `apiAccessTokens` table definition
- `files.ts`: 83 lines different
- `sessions.ts`, `threads.ts`, `messages.ts`: suffix-ID fixes not in templates
- `analytics.ts`: new file, not in templates at all

## Acceptance Criteria

### `agents edit`
```bash
agentforge agents edit <agentId> [--name <n>] [--model <m>] [--description <d>] [--provider <p>] [--active true|false]
# → ✔ Agent updated: agent-test-001
```
Convex: use existing `agents:update` mutation if present, otherwise add it.

### `tokens create`
```bash
agentforge tokens create --name "ci-token" [--expires 2026-12-31]
# → ✔ Token created: agf_xxxx...xxxx
#    Name: ci-token | Expires: 2026-12-31 | Status: Active
```
Convex: use existing `apiAccessTokens:create` mutation.

### `tokens delete`
```bash
agentforge tokens delete <name-or-id> [--force]
# → ✔ Token deleted: ci-token
```
Convex: use existing `apiAccessTokens:remove` mutation. With `--force` skip confirmation.

### Template sync
After all CLI fixes, sync ALL changed convex files to BOTH locations:
```bash
for f in schema.ts sessions.ts threads.ts messages.ts analytics.ts files.ts apiAccessTokens.ts logs.ts usage.ts; do
  [ -f convex/$f ] && cp convex/$f packages/cli/templates/default/convex/$f && echo "synced $f"
done
# Do the same for packages/cli/dist/default/convex/
```
