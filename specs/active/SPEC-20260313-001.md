## PRD (Product Requirements Document)

### Problem Statement
The repo ships two dashboard codepaths (`packages/web` and `packages/cli/templates/default/dashboard`) with inconsistent behavior. The scaffolded dashboard has working project-agent assignment and real file uploads, while `packages/web` still contains stubbed project tabs, placeholder integrations, and hardcoded settings state. Every bug fix and feature lands only once, parity keeps regressing, and E2E confidence remains low.

### User Stories
As a developer using AgentForge
I want a single canonical dashboard
So that features work consistently and I don't encounter broken or stubbed functionality

As a contributor to AgentForge
I want clear guidance on which dashboard is authoritative
So that I don't duplicate effort or introduce drift

### Acceptance Criteria
- [x] Documented canonical dashboard path exists and is referenced from repo docs
- [x] All routes/components reconciled or one copy removed
- [x] Template sync rules updated to include dashboard contract (not just Convex files)
- [x] CI or repo check detects future drift in the canonicalized dashboard surface
- [ ] Follow-up bugs point to canonicalized codepath

### Technical Requirements
- Required APIs: Existing Convex mutations and queries
- Data storage: No changes needed — Convex schema is stable
- Security requirements: No new security surface
