# SPEC-20260304-010: Fix Missing CLI Command Flags & Subcommands

**Status:** Active  
**Priority:** P0 — Broken in v0.10.12  
**Branch:** feat/overnite-a  
**Worktree:** /tmp/agentforge-overnite-a  

## Problem

Several CLI commands are broken or missing in v0.10.12:

1. `agentforge agents create` — missing `--description` flag; the commander definition doesn't expose name/model/description as explicit named options
2. `agentforge sessions delete <id>` — `delete` subcommand completely missing
3. `agentforge workflows create` — `create` subcommand missing (only `list` exists)
4. `agentforge skills install <name>` — crashes with unimplemented error

## Acceptance Criteria

### 1. `agents create` CLI
```bash
agentforge agents create --name "My Agent" --model openai:gpt-4o-mini --description "A helpful agent" --provider openai
# → ✔ Agent created: my-agent
agentforge agents create  # interactive prompts for missing fields
```

### 2. `sessions delete`
```bash
agentforge sessions delete <sessionId>        # prompt for confirm
agentforge sessions delete <sessionId> --force # skip confirm
# → ✔ Session deleted
```
Convex: add `sessions:remove` mutation (soft-delete: set status='deleted', or hard-delete).

### 3. `workflows create`
```bash
agentforge workflows create --name "My Workflow" --agent agent-001 --trigger cron --schedule "*/5 * * * *"
# → ✔ Workflow created: wf-xxxx
```
Convex: reuse existing `workflows:create` mutation if it exists, otherwise add it.

### 4. `skills install`
```bash
agentforge skills install browser-use
agentforge skills install github
# → ✔ Skill installed: browser-use
```
Skill install should: look up the skill in the skillMarketplace table (or a bundled registry), copy SKILL.md to the agentforge project's skills/ directory, and record installation in DB.
