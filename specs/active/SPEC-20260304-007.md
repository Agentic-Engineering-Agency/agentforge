# SPEC-20260304-007: CLI Pagination Fixes

## Status: Active
## Priority: P0

## Objective
Fix two CLI commands that crash with `ArgumentValidationError: missing paginationOpts`.

## Module Overview
- `agentforge logs` calls `api.usage.list` (paginated) without passing `paginationOpts`
- `agentforge threads inspect <id>` calls `api.messages.list` (paginated) without `paginationOpts`
- Both commands need to pass `{ cursor: null, numItems: 50 }` as paginationOpts

## Root Cause
Convex paginated queries require `paginationOpts: { cursor: null | string, numItems: number }`.
The CLI passes `{}` or `{ threadId }` without including the required `paginationOpts` field.

## Requirements
- [ ] `agentforge logs` returns log entries without crashing (uses paginationOpts)
- [ ] `agentforge threads inspect <id>` returns messages without crashing (uses paginationOpts)
- [ ] Both commands handle empty results gracefully
- [ ] CLI source files updated with paginationOpts
- [ ] CLI dist rebuilt: `pnpm build` in packages/cli
- [ ] All tests still pass: `pnpm test` from root
- [ ] Zero TypeScript errors: `tsc --noEmit` in packages/cli

## Technical Details
- File: `packages/cli/src/commands/logs.ts` — find the `api.logs.list` call, add `paginationOpts: { cursor: null, numItems: 50 }`
- File: `packages/cli/src/commands/threads.ts` — find the `api.messages.list` call, add `paginationOpts: { cursor: null, numItems: 50 }`
- The Convex `messages.list` query takes: `{ threadId: id("threads"), paginationOpts: PaginationOptsValidator }`
- The Convex `logs.list` query takes: `{ agentId?, projectId?, paginationOpts: PaginationOptsValidator }`
