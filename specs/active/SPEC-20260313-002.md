## PRD (Product Requirements Document)

### Problem Statement
Project data exists and Convex already exposes `getAllAgents`, `assignAgent`, and `unassignAgent`, but the product surface is inconsistent. The scaffolded dashboard uses those functions, while `packages/web` renders a weaker project detail flow without equivalent assignment behavior. Project settings exist conceptually but don't reliably shape runtime behavior.

### User Stories
As a user managing agents in AgentForge
I want project-agent assignment to work consistently
So that I can organize my agents into projects reliably

As a developer configuring a project
I want project settings to actually influence runtime behavior
So that project-level configuration is meaningful, not decorative

### Acceptance Criteria
- [ ] Project assignment, unassignment, and listing work in canonical dashboard
- [ ] Project configuration uses real Convex state, no placeholder logic
- [ ] Runtime application of project settings is implemented or removed from UI
- [ ] Project-scoped tables/fields documented and validated
- [ ] Tests cover project assignment and settings behavior

### Technical Requirements
- Required APIs: `convex/projects.ts` — `assignAgent`, `unassignAgent`, `getAllAgents`, `getAgents`
- Data storage: Convex projects table — verify schema completeness
- Security requirements: Use `internal.*` not `api.*` for sensitive queries

## Scenarios (Given/When/Then)

### Scenario 1: Assign agent
- **Given** a project exists with no assigned agents
- **When** the user selects an agent and clicks "Assign"
- **Then** the agent appears in the project's agent list immediately

### Scenario 2: Unassign agent
- **Given** a project has 2 assigned agents
- **When** the user unassigns one agent
- **Then** only 1 agent remains in the project's list

### Scenario 3: State persistence
- **Given** an agent was just assigned to a project
- **When** the user refreshes the page
- **Then** the assignment is still visible

### Scenario 4: Empty state
- **Given** a project has no assigned agents
- **When** the user views project configuration
- **Then** a clear empty state with assignment CTA is shown

## Technical Approach (HOW)

### Architecture
- Data layer: `convex/projects.ts` — single source of truth
- Dashboard: canonical dashboard path (per SPEC-20260313-001)
- Runtime: `packages/runtime/` reads project config on agent initialization
- Precedence: project settings < agent defaults < thread-level overrides
