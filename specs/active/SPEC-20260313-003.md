## PRD (Product Requirements Document)

### Problem Statement
Users cannot select agents from the agent list in project configuration. This is a confirmed product bug. Convex already exposes the required assignment mutations, and the scaffolded dashboard demonstrates the intended behavior, but the currently exercised dashboard path fails to provide a usable agent-selection flow.

### User Stories
As a user configuring a project
I want to assign agents from the available agent list
So that my project has the right agents working on it

As a user managing project agents
I want to unassign agents I no longer need
So that my project configuration stays clean

### Acceptance Criteria
- [ ] Users can open project configuration and assign an unassigned agent
- [ ] Users can unassign an assigned agent
- [ ] UI reflects assignment state immediately and survives refresh
- [ ] Empty-state and confirmation behavior are clear
- [ ] Bug is covered by browser-driven tests

### Technical Requirements
- Required APIs: Existing `convex/projects.ts` mutations (no backend changes needed)
- Data storage: No schema changes needed
- Security requirements: Mutations should use proper authorization checks

## Scenarios (Given/When/Then)

### Scenario 1: Assign agent
- **Given** a project detail page is open with available unassigned agents
- **When** the user clicks "Assign" on an agent
- **Then** the agent moves to the assigned list immediately

### Scenario 2: Unassign agent
- **Given** a project has assigned agents displayed
- **When** the user clicks "Unassign" on an assigned agent
- **Then** the agent is removed from the assigned list immediately

### Scenario 3: No available agents
- **Given** all agents are already assigned to the project
- **When** the user views the assignment panel
- **Then** a message "All agents are assigned" is shown

### Scenario 4: Reproduce the bug
